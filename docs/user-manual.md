# MD Viewer 使用手册

本手册面向日常使用者，说明 MD Viewer 的主要工作流、导出边界和 v2.3.0 已验证的稳定性范围。

## 快速上手

1. 启动 MD Viewer。
2. 点击“打开文件夹”，选择包含 Markdown 文件的目录。
3. 在左侧文件树中点击 `.md` 文件，右侧默认以预览模式打开。
4. 使用顶部搜索或 `Cmd+K` / `Ctrl+K` 查找文件。
5. 需要修改文档时切换到编辑模式；不编辑时保持默认预览即可。

## 文件树与搜索

左侧文件树显示当前目录下的 Markdown 文档和支持的图表文件。文件树过滤框常驻显示，可输入中文、英文、文件名片段或路径片段。清空过滤词后，文件列表恢复到完整状态。

文件树支持最近文件夹、最近文件和历史菜单。v2.3.0 已重点验证滚动后的文件点击、历史菜单显示边界和过滤框焦点，避免点击文件后滚动位置回到顶部。

## 预览与编辑模式

MD Viewer 以预览为默认模式。进入编辑模式后，可以使用两种方式修改 Markdown：

- 源码编辑：适合复杂结构、图表代码块、长表格和精确 Markdown 语法。
- 渲染区直接编辑：适合段落、标题、列表、引用、普通代码块和表格单元格等常见块。

编辑模式会保护未保存草稿。退出编辑、切换文件或关闭相关标签时，如果存在未保存内容，会出现确认提示。选择放弃后回到磁盘版本，选择取消则保留草稿。

## 图表与公式

MD Viewer 支持 Mermaid、ECharts、Markmap、Graphviz、PlantUML、DrawIO、Infographic、KaTeX、Excalidraw、Vega-Lite、D2、BPMN、WaveDrom、C4-PlantUML、Structurizr、Plotly、DBML、AntV G6、Kroki 等图表或公式。

多数图表支持源码查看、全屏、缩放、适应大小和下载图片。v2.3.0 已统一多类图表的全屏遮罩、右上角关闭按钮、底部工具栏和 Escape 退出行为。

## 图表打包下载

在 Markdown 预览区右键，可使用“打包下载图表”导出当前文档中的图表图片。导出的 ZIP 会创建一个以 Markdown 文件名命名的顶层文件夹，图片按文档顺序命名，例如 `01-mermaid.png`、`02-d2.png`。

导出图片会尽量保持图表比例和内容居中。对于极宽图表，v2.3.0 使用白底补边方式限制异常宽高比，避免图表内容被拉伸变形。

## HTML、PDF 与 DOCX 导出

HTML 和 PDF 导出由桌面端完成，适合离线保存当前预览效果。v2.3.0 已验证导出的 HTML 包含 Markdown 核心样式，PDF 输出不是空白文件，并覆盖特殊字符文件名。

DOCX 导出入口在桌面端内置，但高质量 Word 生成依赖可选的外部 `md-viewer-docx-service`。如果未配置服务地址，DOCX 服务相关测试会跳过并记录原因；如果已配置服务，服务不可达、版本过低或样式不支持时会显示更明确的诊断。

## 稳定性与限制

v2.3.0 是稳定性与一致性版本，不新增图表类型，也不改变 MD Viewer 预览优先的定位。

已重点验证：

- 文件树过滤、中文输入、滚动保持和历史菜单。
- 编辑模式退出、放弃草稿、保存后滚动保持、同步滚动和分屏编辑。
- 图表全屏、关闭按钮、Escape 退出、工具栏操作和退出后原图保留。
- HTML/PDF 导出、图表 PNG 下载和图表打包导出。

仍需注意：

- 渲染区直接编辑不适合复杂图表、嵌入式组件和非标准 Markdown，建议使用源码编辑。
- PlantUML、Kroki、DOCX 服务等能力可能依赖网络、外部服务或本机部署。
- DOCX 导出不是浏览器预览的像素级复制，正文会映射为 Word 样式，图表通常以截图方式注入。

## 本地验证命令

开发或发版前可运行：

```bash
npm run typecheck
npm run lint
npm test -- --run
npm run build
scripts/release-smoke.sh quick
scripts/release-smoke.sh full
```

`quick` 用于日常修复后的快速门禁，`full` 用于发版前完整门禁。
